# Руководство по внесению вклада (CONTRIBUTE)

Добро пожаловать в проект **AriaSignature-Cfe**! Данное руководство содержит стандарты разработки и правила внесения изменений для разработчиков и контрибьюторов расширения 1С и интеграционного контура Melezh.

---

## 1. Общие принципы разработки

Мы следуем правилу: **выбирать решения, которые обеспечат максимально простое обновление конфигурации в будущем**, даже если они требуют больше времени на реализацию.

### Ключевые требования:
1. **Префиксы объектов**: Все новые объекты метаданных, реквизиты, общие модули и роли должны создаваться с префиксом `АС_` (например, `АС_КонтрагентыAria`).
2. **Суффиксы синонимов**: Синонимы для всех добавленных объектов должны заканчиваться на ` (АС)`.
3. **Разработка форм**: Изменения в формах производятся **только интерактивно** в конфигураторе 1С. Программное конструирование/модификация форм запрещены.
4. **Без БСП и внешних библиотек**: В расширении не используются функции БСП или сторонние библиотеки форм как источники бизнес-логики.
5. **Принцип записи данных**: Запись данных в регистры сведений выполняется **строго по схеме: Документ -> Движения регистра**. Прямая запись в регистры из обработок или внешних источников не допускается.

---

## 2. Правила оформления кода

Весь измененный или добавленный код должен быть четко выделен тегами с указанием автора, даты и номера листа требований (ЛТ).

### Шаблон добавления нового кода:
```bsl
//+АС Бармин А.С. 2026-05-24 : ЛТ-03 Реализация HTTP-клиента Melezh
// Здесь располагается новый программный код
//-АС Бармин А.С. 2026-05-24
```

### Шаблон изменения существующего кода (документирование «было/стало»):
```bsl
//+АС Бармин А.С. 2026-05-24 : ЛТ-04 Исправление обработки ошибок
//было
//Результат = ВыполнитьЗапросКMelezh();
//стало
Результат = АС_ИнтеграцияMelezhСлужебный.ВыполнитьБезопасныйЗапрос(Компьютер);
//-АС Бармин А.С. 2026-05-24
```

## 3. Архитектура интеграции

Интеграционный контур расширения разделен на два уровня:

1. **Основной контур (Melezh Proxy)**:
   - Вся бизнес-логика 1С работает через службу `Melezh` (`AriaSignatureMelezhService`) на порту **`7788`**.
   - Адрес шлюза: `http://<host>:7788/<handler_key>`.
   - Основные методы: `aria_get_disks`, `aria_get_backups`, `aria_get_backups_logs` и т.д.
2. **Диагностический контур (AriaSignature API)**:
   - Прямой вызов API AriaSignature на порту **`5160`** (`:5160/api/v1/*`) допускается **исключительно** для интерактивных проверок оператором или локальной диагностики. Использовать его для автоматического сбора данных запрещено.

---

## 4. Ролевая модель

При добавлении новых объектов метаданных обязательно создавайте **две роли**:
* `АС_<ИмяОбъекта>_Чтение` — только чтение и просмотр.
* `АС_<ИмяОбъекта>_Изменение` — создание, изменение, проведение и выполнение действий.

---

## 5. Работа с Хранилищем конфигурации 1С

Групповая разработка расширения ведется через стандартное Хранилище 1С.

### Памятка разработчика:
* Перед добавлением новых объектов обязательно **захватите корень** конфигурации расширения. Поместите его обратно сразу после создания объекта.
* При редактировании объектов выполняйте **рекурсивный захват** формы/модуля, чтобы избежать коллизий с другими разработчиками.
* Всегда пишите содержательные комментарии при помещении изменений в хранилище (номер ЛТ, краткое описание сделанного).
* Для облегчения подключения имя пользователя в хранилище совпадает с именем вашей локальной базы, пароль оставляется пустым.
